# 项目整体

## 1. 项目是什么

这是一个基于 `Vue 3 + Vite` 的单页应用（SPA），用于做每日量化记录与统计。

- 录入页：按日期填写英语、编程、可选项，自动计分
- 统计页：粘贴多天导出文本，生成趋势图
- 离线能力：支持本地离线构建和桌面“类应用窗口”启动

当前技术栈很轻量：

- 视图框架：`vue`
- 构建工具：`vite`
- 图表：`chart.js`

---

## 2. 先会跑起来（使用方法）

在项目根目录执行：

```bash
npm install
npm run dev
```

常用命令：

- `npm run dev`：开发模式（热更新）
- `npm run build`：生产构建（GitHub Pages 口径）
- `npm run build:offline`：离线构建（资源相对路径，适合本地）
- `npm run preview`：本地预览构建产物

Windows 离线启动脚本：

- `start-offline.bat`：构建离线包 + 启动本地预览服务
- `start-offline-silent.vbs`：静默启动，并以独立窗口打开（600x700 居中）

## 4. 每个关键文件/文件夹做什么

### `src/main.js`

应用入口文件，职责：

- 引入全局样式 `src/style.css`
- 挂载根组件 `App.vue`
- 在生产环境监听 `vite:preloadError`，分包缓存异常时自动刷新

> 学习点：Vite 项目入口通常非常薄，只负责“启动应用”。

### `src/App.vue`

主页面与主交互中心，职责：

- 管理“录入页 / 统计页”切换
- 展示录入表单、分数卡片、目标达成、导入导出按钮
- 生成“总结并复制”文本
- 处理导入文本并写回状态
- 调用 `useProgressStore` 获取状态与分数

> 学习点：这个文件是“页面编排层 + 交互层”，适合先通读它理解业务全貌。

### `src/stores/useProgressStore.js`

业务状态与计分规则核心，职责：

- 定义默认状态（各输入字段）
- 做输入归一（非负整数）
- 兼容旧字段导入（例如旧版技术难点/优化细节字段）
- 统一计算英语、编程、可选、总分
- 提供 `reset()` 重置能力

> 学习点：这是整个项目的“规则引擎”，复现项目时建议优先手写这一层。

### `src/components/StatsView.vue`

统计分析页面，职责：

- 解析多天文本记录
- 按同一套计分规则换算数据
- 用 `Chart.js` 渲染趋势图
- 支持图表复制到剪贴板

> 学习点：它展示了“文本协议 -> 指标计算 -> 可视化”的完整链路。

### `src/style.css`

全局样式（字体、背景、基础布局等）。  
`App.vue` 与 `StatsView.vue` 里还有大量 `scoped` 局部样式，负责组件内部 UI。

### `index.html`

Vite 的 HTML 入口模板。核心是一个 `#app` 节点和 `src/main.js` 脚本引入。

### `vite.config.js`

Vite 配置核心点：

- 启用 Vue 插件
- 根据 `mode` 切换 `base`
  - 默认：`/quant-dashboard/`（GitHub Pages 子路径）
  - `offline`：`./`（离线相对路径）

### `package.json`

定义项目依赖与脚本：

- 运行命令（`dev/build/build:offline/preview`）
- 运行时依赖（`vue/chart.js`）
- 开发依赖（`vite/@vitejs/plugin-vue`）

### `start-offline.bat`

Windows 下离线运行辅助脚本：

1. 检查 npm 是否可用
2. 执行 `npm run build:offline`
3. 启动本地预览服务 `vite preview --host --port 4173`
4. 打开 `http://localhost:4173/quant-dashboard/`

### `start-offline-silent.vbs`

静默启动脚本，职责：

- 隐藏执行 `start-offline.bat --no-open`
- 等待服务启动后，优先用 Edge/Chrome `--app` 模式开独立窗口
- 默认窗口大小 `600x700`，居中显示

### `.github/workflows/deploy.yml`

GitHub Actions 部署配置（用于自动化部署流程）。

### `.gitignore`

定义 Git 忽略规则（`node_modules`、`dist`、日志等）。

## 5. 文件之间怎么协作（调用链）

最核心链路可以记成：

`index.html` -> `main.js` -> `App.vue` -> `useProgressStore.js`（计分）  
`App.vue` -> `StatsView.vue`（统计页） -> `Chart.js`（画图）

导入/导出链路：

- 导出：`App.vue` 把状态序列化为文本并复制
- 导入：`App.vue` 解析文本，做字段兼容与上限修正
- 统计：`StatsView.vue` 读取同类文本并独立计算曲线指标

---

## 6. 你要“手写复现”时建议顺序

1. 先搭最小 Vite + Vue 骨架（`main.js` + `App.vue`）
2. 写 `useProgressStore.js`（默认状态 + 计分函数）
3. 在 `App.vue` 完成录入表单和分数卡片
4. 加导出/导入文本协议
5. 加 `StatsView.vue`（先解析文本，再接图表）
6. 最后补 `build:offline` 与离线脚本

这样做的好处是：每一步都能运行、都能验证，出错时定位最清晰。

---

## 7. 新手容易踩的点

- `vite.config.js` 的 `base` 不对，会导致离线白屏
- 录入页和统计页必须用同一计分口径，否则数据会“看起来不一致”
- 导入兼容旧字段时，要做“新旧字段映射 + 上限限制”
- `file://` 场景下某些浏览器 API（如剪贴板）可能受限，建议走本地预览服务

---

## 8. 一句话总结

这个项目是一个“规则驱动的前端工具”：  
`store` 负责规则，`App` 负责录入交互，`StatsView` 负责统计可视化，`Vite` 负责构建与部署。
