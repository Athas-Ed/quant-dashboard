# 我用 Vue3 手写了一个可离线的每日量化看板（1）：项目背景与目标拆解

## 摘要

这篇是系列第一篇，不讲细节实现，先讲清楚三个问题：为什么做、做成什么、为什么这样选技术。  
目标是把一个“个人记录需求”拆成可执行的产品最小闭环（MVP），并给后续 5 篇实现文章打底。

---

## 1. 我为什么要做这个项目

我需要一个每天都能用的轻量工具，用来记录学习与创作投入，并把投入转成可比较的分数。  
现成工具要么太重（项目管理软件），要么规则不贴合（通用打卡工具）。

我真正需要的是：

- 输入快：当天打开就能填
- 规则明确：每项分值和上限透明
- 可复盘：能看趋势，不只看当天
- 可迁移：支持导入导出，不怕换设备
- 可离线：断网也能用

---

## 2. 我把需求拆成了什么

我把功能拆成两个页面：

1. **录入页**：每天填数据 + 自动计分 + 生成总结
2. **统计页**：粘贴多天数据 + 生成趋势图

再拆成三层职责：

- **规则层**：所有计分规则统一在 store 中计算
- **交互层**：录入页负责表单、导入导出、复制总结
- **展示层**：统计页负责解析文本和图表渲染

这样拆的好处是：后续改规则时，不会在多个页面到处改。

---

## 3. 规则口径（先定标准，再写代码）

我当前采用的是：

- 英语上限：50
- 编程上限：70
- 可选上限：60
- 总分上限：180

核心原则：

- 分项可有上限
- 总项也要有上限
- 导入数据必须归一（非负整数、异常兜底）

先把规则写清楚，再进入实现，能减少大量返工。

---

## 4. 为什么选 Vue3 + Vite + Chart.js

### Vue3

- 组合式 API 非常适合“规则函数 + 页面交互”这类工具型项目
- `ref/reactive/computed` 足够表达业务状态

### Vite

- 本地启动快，改动反馈快
- 构建配置简单，适合快速迭代
- 能用 `mode` 做线上/离线构建区分

### Chart.js

- 学习成本低
- 折线图配置直观
- 对这种“日趋势”场景够用

---

## 5. MVP 边界怎么控制

我刻意不做这些：

- 用户系统
- 云同步
- 后端服务
- 复杂权限

只做本地可用闭环：

- 录入 -> 计分 -> 导出 -> 导入 -> 统计 -> 离线运行

这能保证项目短周期做出价值，而不是陷进“功能永远做不完”。

---

## 6. 这一篇最重要的结论

对于个人工具项目，真正要先做的不是 UI，而是：

1. 规则口径统一
2. 数据协议可迁移
3. 功能边界可收敛

把这三件事先定好，后面的实现会非常顺。
